# コミット&プッシュ

変更をコミットし、GitHub CLIを用いてpushしてください。

## コミットメッセージのルール

- コミットメッセージは日本語で作成すること。
- 読みやすいように箇条書きなどを活用すること。
- 1行目は変更の要約（件名）、空行を挟んで詳細（本文）を記載すること。

## コミット対象

- 変更内容を確認し、関連する変更をまとめてステージングすること。
- コミット対象がなければ、pushのみ行うこと。
- worktree内で作業中の場合は、それも考慮すること。

## 改行を含むコミットメッセージの作成方法

`git commit -m "…\n…"` のように、`-m` に渡す文字列の中で `\n` を書いても、実際の改行にはならず、`\n` という文字列がそのまま表示されてしまいます。

以下のいずれかの方法を使用してください：

### 方法1: ヒアドキュメントを使用

```bash
git commit -F - <<EOF
件名: 変更の要約

- 変更点1
- 変更点2
EOF
```

### 方法2: 複数の `-m` オプションを使用

```bash
git commit -m "件名: 変更の要約" -m "- 変更点1" -m "- 変更点2"
```

### 方法3: $'...' 形式を使用（bash）

```bash
git commit -m $'件名: 変更の要約\n\n- 変更点1\n- 変更点2'
```
