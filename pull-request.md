# プルリクエスト

GitHub CLIを用いて、プルリクエストを作成してください。

## プルリクエストのタイトルと本文

- 日本語で記載すること。
- 読みやすいように箇条書きなどを活用すること。
- 変更内容が具体的に伝わる内容（概要・主な修正点・テスト状況など）を整理して記載すること。

## ブランチの確認

- 現在のブランチとベースブランチ（通常は `main` または `develop`）を確認すること。
- 必要に応じて `--base` オプションでベースブランチを指定すること。

## 改行を含む本文の作成方法

`gh pr create` で本文に改行を含める場合、以下の方法を使用してください：

### 方法1: ヒアドキュメントを使用（推奨）

```bash
gh pr create --title "PRタイトル" --body "$(cat <<EOF
## 概要
変更の概要を記載

## 主な変更点
- 変更点1
- 変更点2

## テスト状況
- テスト内容を記載
EOF
)"
```

### 方法2: `--body-file` オプションを使用

一時ファイルに本文を書き出してから使用：

```bash
gh pr create --title "PRタイトル" --body-file pr-body.md
```

## ドラフトPR

作業途中でレビュー依頼前にPRを作成したい場合は、`--draft` オプションを使用：

```bash
gh pr create --draft --title "WIP: PRタイトル" --body "..."
```
